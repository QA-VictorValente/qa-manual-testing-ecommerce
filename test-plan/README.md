# Test Plan - E-commerce QA Project

## 🎯 1. Objetivo
Este documento descreve o plano de testes para validação de uma aplicação fictícia de e-commerce, garantindo que as principais funcionalidades estejam funcionando corretamente e atendam às regras de negócio.

---

## 📦 2. Escopo

### Funcionalidades incluídas:
- Login de usuário
- Cadastro de usuário
- Pesquisa de produtos
- Adição de produtos ao carrinho
- Finalização de compra

### Funcionalidades não incluídas:
- Performance
- Segurança avançada
- Testes de carga
- Integrações externas

---

## 🧪 3. Tipos de Teste

- Testes funcionais
- Testes exploratórios
- Testes de regressão
- Testes de usabilidade (básico)

---

## 🚪 4. Critérios de Entrada

- Ambiente de teste disponível
- Aplicação funcional
- Dados de teste preparados
- Requisitos básicos definidos

---

## 🚪 5. Critérios de Saída

- Todos os casos de teste executados
- Bugs críticos reportados
- Funcionalidades principais validadas
- Sem falhas bloqueadoras abertas

---

## 🌍 6. Ambiente de Teste

- Navegador: Chrome / Edge
- Sistema: Web
- Dispositivo: Desktop
- Internet estável
